AI-First Rhythm Generation — 調査レポート 2026-04-06

1. 現状の問題

現在のリズム生成は rhythm-engine.ts の11個の固定パターンに基づいており、人工感が強い。特にアルペジオ（arpUp/Down）は毎拍まったく同じタイミング（0, 120, 240, 360 ticks）で機械的。

パターン特徴問題
arpUp/Down低→高/高→低の固定分割毎拍完全に同じタイミング
bossa-novabeat偶奇で bass/upper 交互パターンが硬く固定
comp-jazzCharleston風の3ヒット位置が固定
block全音同時単調

@humanize と @swing は後処理で乱数を加算するが、パターン自体の「個性」はコントロールできない。また Math.random() を使っているため再現性がない。

2. 検討した4つのアプローチ

案A: 現状延長

AIは既存パターン名のみ提案。人工感は消えない。効果なし

案B: AIがカスタムパターン生成

AIがPatternHit[]をJSON的に出力。自由度は高いがAI出力制御が複雑。1.5-2週間

案C: AIがMIDI直接生成

決定論的エンジンの原則に矛盾。再現性なし。テキストファースト破壊。非推奨

案D: ハイブリッド (推奨)

AIがパラメータ + humanize-seed を提案。既存枠組み内で最大限の個別化。1-1.5週間

3. 推奨案: ハイブリッド (案D)

コアアイデア: @humanize-seed ディレクティブの導入。シード値で再現可能な「揺れ」を生成。AIが曲に合ったシード値を提案する。

ユーザーフロー

  1. BlockPopover で「ジャズバラード風」と入力
  2. AI が以下を一括生成: 
    @mode: arpUp
@voicing: close
@velocity: p→mf
@humanize: 18%
@swing: 35%
@strum: 6ms
@humanize-seed: 4276
  3. MIDIプレビューで即座に確認
  4. 「もう少しルーズに」→ humanize を 20% に微調整
  5. OK で確定

humanize-seed の技術詳細

// 線形合同法による決定論的乱数
function seededRandom(seed: number): () => number {
  let state = seed;
  return () => {
    state = (state * 1103515245 + 12345) & 0x7fffffff;
    return state / 0x7fffffff;
  };
}

// 同じseed → 常に同じ「揺れ方」
// seed を変える → 異なる「揺れ方」（パターンは同じ）

現在のBlockPopoverはスライダーが前面に出すぎている。「パラメータを知らなくても使える」UIに転換。

7. 結論

案D（ハイブリッド）を推奨

  • 1〜1.5週間で完成可能
  • テキストファースト・決定論的エンジンの原則を守る
  • 再現性を完全に保つ（humanize-seed）
  • 既存パターンの枠内で最大限の個別化
  • UIは「AI入力 → プレビュー → 微調整」の自然なフロー
